# 🐂 Elite Fitness Academy
### نظام إدارة الجيم واللاعبين (Gym & Athlete Management System)

نظام ويب متكامل لإدارة أكاديمية اللياقة البدنية: اللاعبين، الفروع، القياسات، الحضور والغياب، والتقارير — كل ده بدل الورق وملفات الإكسل المتفرقة.

مخصص لمدرب واحد فقط (Single coach) عنده Login واحد، وكل البيانات بتتخزن بشكل دائم على الجهاز اللي شغال عليه السيستم.

---

## 🧱 التقنيات المستخدمة

| الجزء | التقنية |
|---|---|
| الواجهة (Frontend) | React + Vite + Tailwind CSS |
| السيرفر (Backend) | Node.js + Express |
| قاعدة البيانات | ملف JSON محلي (بدون الحاجة لتثبيت أي قاعدة بيانات منفصلة) |
| التقارير | ExcelJS (ملفات Excel) + pdfmake (ملفات PDF بالعربي بشكل صحيح) |
| المصادقة | JWT + كلمة مرور مشفّرة (bcrypt) |

> ✅ لا حاجة لتثبيت MySQL أو PostgreSQL أو أي سيرفر قاعدة بيانات. النظام يخزن البيانات في ملف واحد بسيط (`server/src/data/db.json`) وهو يكفي تمامًا لمدرب واحد وعدد لاعبين محدود.

## 🚀 التشغيل لأول مرة

### المتطلبات
- تثبيت [Node.js](https://nodejs.org) (نسخة 18 أو أحدث) على الجهاز.

### الخطوة 1: تثبيت حزم السيرفر

```bash
cd server
npm install
```

### الخطوة 2: إنشاء حساب المدرب (مرة واحدة فقط)

```bash
npm run seed -- --username=coach --password=ضع_كلمة_مرور_قوية --name="اسمك"
```

> غيّر `username` و `password` و `name` للقيم اللي تحبها. لو مش حددت حاجة، هيستخدم قيم افتراضية (username: coach, password: elite2009).

### الخطوة 3: تشغيل السيرفر

```bash
npm run dev
```

السيرفر هيشتغل على: `http://localhost:4000`

### الخطوة 4: تثبيت وتشغيل الواجهة (في نافذة Terminal تانية)

```bash
cd client
npm install
npm run dev
```

الواجهة هتفتح على: `http://localhost:5173`

افتح الرابط ده في المتصفح وسجّل دخول بالبيانات اللي أنشأتها في الخطوة 2. 🎉

## 🔐 الأمان وحماية البيانات

- في حساب مدرب واحد بس (Single coach) — مفيش تسجيل حسابات تانية من الواجهة.
- كلمة المرور مشفّرة بالكامل ولا تُحفظ كنص عادي.
- تقدر تغيّر كلمة المرور في أي وقت من صفحة **الإعدادات** داخل النظام.
- كل بيانات اللاعبين محفوظة في ملف `server/src/data/db.json` — اعمل نسخة احتياطية (backup) من هذا الملف بشكل دوري (ممكن تنسخه على فلاشة أو جوجل درايف).

## 📊 المميزات

- **إدارة اللاعبين**: إضافة، تعديل، حذف، بحث، فلترة حسب الفرع. بيانات كل لاعب تشمل تاريخ الميلاد (والسن يُحسب تلقائيًا منه ويتحدث من نفسه كل سنة)، والنادي أو الأكاديمية الأصلية اللي اللاعب جاي منها (منفصلة تمامًا عن فرع أكاديميتك).
- **إدارة الفروع**: عدد غير محدود من الفروع، نقل لاعب من فرع لآخر مع الاحتفاظ بكل بياناته وسجله.
- **القياسات**: حدّد بنفسك أي تمارين أو قياسات تحب تتابعها (مش محدودة بقائمة ثابتة) من صفحة **أنواع القياسات** — تقدر تضيف، تعدّل، أو تحذف أي قياس، وتحدد وحدته (ثانية، سم، م...)، وتقدر كمان تحدد لو "الأقل هو الأفضل" بالنسبة له (مفيد لقياسات الزمن زي عداء 100 متر، حيث الأقل وقتًا = أداء أفضل). القياسات اللي تضيفها تظهر تلقائيًا في نموذج تسجيل القياسات وفي كل التقارير.
- **متابعة التطور**: مقارنة آلية بين آخر قياسين، ورسم بياني لتطور الأداء عبر الزمن.
- **الحضور والغياب**: تسجيل حضور يومي لكل فرع، حساب نسبة الالتزام لكل لاعب وكل فرع.
- **لوحة تحكم**: نظرة شاملة سريعة على كل الأرقام المهمة، وبحث سريع عن أي لاعب.
- **التقارير**: تصدير أي بيانات كملف Excel أو PDF بالعربي بشكل صحيح (دعم كامل لـ RTL).
- **النسخ الاحتياطي**: من صفحة الإعدادات، تقدر تنزّل نسخة احتياطية كاملة من كل بيانات النظام في ملف واحد، وتسترجعها وقتما تحب (مثلاً لو غيّرت جهاز أو حصل أي مشكلة). احتفظ بنسخ دورية في مكان آمن (فلاشة أو جوجل درايف).

## 🖥️ تشغيل النظام يوميًا (بعد أول مرة)

كل مرة تيجي تشغّل النظام، تحتاج تفتح نافذتين Terminal:

**نافذة 1 (السيرفر):**
```bash
cd server
npm run dev
```

**نافذة 2 (الواجهة):**
```bash
cd client
npm run dev
```

بعد كذا، افتح `http://localhost:5173` في المتصفح.

> 💡 نصيحة: تقدر تسيب السيرفر شغال طول الوقت على نفس الجهاز (مثلاً جهاز مكتبي في الجيم) وتدخل عليه من أي متصفح على نفس الشبكة.

## 🛠️ استكشاف الأخطاء

| المشكلة | الحل |
|---|---|
| "لا يوجد حساب مدرب مسجّل" | شغّل `npm run seed` داخل مجلد `server` |
| الواجهة مش بتتصل بالسيرفر | تأكد إن السيرفر شغال على بورت 4000 قبل ما تفتح الواجهة |
| نسيت كلمة المرور | احذف ملف `server/src/data/db.json` وشغّل `npm run seed` تاني (⚠️ هيمسح كل البيانات، فاعمل نسخة احتياطية أولاً) |
| لو بتحدّث نسخة قديمة من النظام ومحتاج "أنواع القياسات" الافتراضية تظهر | متقلقش، أول ما تشغّل السيرفر (`npm run dev`) هيضيفها تلقائيًا بدون أي أمر إضافي |
